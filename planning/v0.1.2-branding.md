# v0.1.2 - Brand Identity: "Formateur"

**Status:** âœ… Implemented
**Goal:** Establish a unique, premium visual identity for the game, moving away from the generic "Vite+React" look.

## 1. The Boardroom (Alignment Phase)

**Attendees:** Product Manager, Frontend Specialist, Belgian Politics Expert.

*   **Product Manager:** "We need a name that screams 'Belgian Politics' but feels high-end. No more 'BelPolSim'."
*   **Politics Expert:** "The most prestigious role is the *Formateur*. It's the architect of the government. It fits our gameplay loop perfectly."
*   **Frontend Specialist:** "I love it. For the visual style, I'm thinking 'Elegant Bureaucracy'. Dark, serious backgrounds (Charcoal) with rich accents (Royal Gold) and clean, paper-like cards (Cream). We need to drop the generic Slate/Blue look."
*   **Consensus:**
    *   **Name:** Formateur
    *   **Vibe:** The Economist meets Civilization.
    *   **Action:** Refactor the shell (Sidebar, TopBar, Layout) to match this new identity immediately.

## 2. Implementation Details

### 2.1. Brand Assets
*   **Logo:** Created and stored in `brand-design/logo.png`.
*   **Style Guide:** Defined in `brand-design/style-guide.md`.

### 2.2. Codebase Changes
*   **`index.html`:** Title updated to "Formateur".
*   **`src/index.css`:**
    *   Imported **Playfair Display** (Serif) for headings and **Inter** (Sans) for UI.
    *   Defined CSS variables for the new palette:
        *   `--color-cabinet-charcoal` (#1A202C)
        *   `--color-royal-gold` (#FDC12A)
        *   `--color-bureaucratic-cream` (#F7FAFC)
    *   Created semantic classes: `.card`, `.btn-primary`, `.btn-danger`.
*   **`src/components/Layout.tsx`:** Removed hardcoded Slate backgrounds.
*   **`src/components/Sidebar.tsx`:**
    *   Updated branding to "FORMATEUR".
    *   Applied Charcoal background and Gold accents.
*   **`src/components/TopBar.tsx`:**
    *   Applied Charcoal background.
    *   Fixed layout structure.
*   **`src/components/GameView.tsx`:**
    *   Refactored main containers to use `.card`.
    *   Updated buttons to use `.btn-primary` and `.btn-danger`.

## 3. Next Steps (Future Refactoring)
*   **Dashboards:** `CampaignDashboard` and `PollingDashboard` still use hardcoded Slate styles. They need to be refactored to use the new `.card` and color variables.
*   **Map:** The map colors should be adjusted to fit the new palette (Gold/Charcoal).
